## Описание

Приложение предоставляющее список тренировок, с возможносью поиска и фильтрации и просмотра деталей
тренировки.
Так же, предоставляет видео плеер, с доступной перемоткой, выбором скорости воспроизведения по
умолчанию и возможносью выбора качества.

## Модули

Проект разбит на 5 модулей:

```
:app                          # Содержит Application, DI-модуль, граф навигации (Директории /workouts и /workoutsdetails намеренно лежат в :app, для упрощения работы с навигацией)
:core:network                 # Содержит основные сущности для сетевого взаимодействия
:core:coroutines              # Модуль для работы с корутинами
:feature:player:api           # API видеоплеера
:feature:player:impl          # Реализация видеоплеера
```

## Навигация

Single-Activity (`MainActivity`) + `NavHostFragment` с двумя экранами:

```
WorkoutsFragment  ──(action)──►  WorkoutDetailsFragment
   (список)                       (детали + плеер)
```

## Примечания

1) Параметр `duration` в ответе от /get_video?id и /get_workouts приходит разный - поэтому, было
   решено отображать то, что пришло в ответе, как есть 
2) Если `duration` не придет или будет пустым или невалидным - тренировка отобразится, но, длительность отображаться не будет
3) Если не придет `id` или `title`, то тренировка отображаться не будет
4) Варианты состояния экранов `ClientError` и `ServerError` в UI-слое, не являются данными из `data` слоя. Это вариант состояния, при котором по-разному происходит обработка сетвых ошибок. Например, если в ответе мы получаем 404-ую ошибку, то на экране отображаем текст "Что-то пошло не так", если 500-ую, то текст "Что-то пошло не так" и кнопку для повторного запроса
5) При повторной попытке загрузки данных на главном экране сохраняются раннее введеный поисковой запрос и выбранный фильтр
6) На главном экране реализован сброс фильтров и очистка поискового запроса
7) На экране деталей тренировки в плеере, есть возможность выбора качества воспроизведения, однако варинаты выбора зависят от видео-дорожки
8) Приложение поддерживает ночной режим и увеличенный системный шрифт
9) При смене конфигурации плеер останавливается, сохраняет состояние воспроизведения
10) Плеер сохраняет состояние воспроизведения при выходе из деталей тренировки и повторном возврате
